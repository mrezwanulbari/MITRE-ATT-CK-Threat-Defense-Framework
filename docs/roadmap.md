# MITRE ATT&CK Threat Defense Framework — Roadmap

This roadmap outlines planned enhancements to expand coverage, improve detection quality, and strengthen SOC operationalization.

---

## Phase 1 — Detection Expansion
- Add more Sigma rules (Windows, Linux, Cloud)
- Add more YARA rules (ransomware families, webshell variants)
- Expand KQL hunting queries for identity and cloud

## Phase 2 — ATT&CK Coverage Enhancement
- Expand enterprise.json and cloud.json mappings
- Add technique-to-playbook mapping automation
- Improve coverage-matrix.md with scoring

## Phase 3 — SOC Operationalization
- Add more SOC playbooks (identity, cloud, endpoint)
- Add incident simulation guides
- Add purple-team exercise templates

## Phase 4 — Cloud Defense Integration
- Add Azure/AWS detection bundles
- Add cloud identity attack patterns
- Add cloud-specific ATT&CK mappings

## Phase 5 — Automation & Tooling
- Add scripts for mapping validation
- Add detection testing automation
- Add CI/CD pipeline for rule validation

## Phase 6 — Documentation & Architecture
- Add architecture diagrams
- Add maturity model enhancements
- Add implementation case studies

---

This roadmap ensures continuous improvement and alignment with evolving threats.
